Part A.2: Recover Homographies

Homography Estimation (DLT)

We estimate a projective transform H that maps source points \((x,y)\) (from im2_pts) to target points \((x',y')\) (from im1_pts):

$$\begin{aligned} x' &= \frac{h_1 x + h_2 y + h_3}{h_7 x + h_8 y + h_9}, \\ y' &= \frac{h_4 x + h_5 y + h_6}{h_7 x + h_8 y + h_9}. \end{aligned}$$

Rearranging gives two linear equations per correspondence:

$$x'(h_7 x + h_8 y + h_9) = h_1 x + h_2 y + h_3,$$ $$y'(h_7 x + h_8 y + h_9) = h_4 x + h_5 y + h_6.$$

Stacking the equations for \(n\) correspondences yields a homogeneous system \(\mathbf{A}\mathbf{h}= \mathbf{0}\) with \(\mathbf{h}=[h_1,\dots,h_9]^T\) and \(\mathbf{A}\in\mathbb{R}^{2n\times 9}\):

$$\mathbf{A} = \begin{bmatrix} x_1 & y_1 & 1 & 0 & 0 & 0 & -x'_1 x_1 & -x'_1 y_1 & -x'_1 \\ 0 & 0 & 0 & x_1 & y_1 & 1 & -y'_1 x_1 & -y'_1 y_1 & -y'_1 \\ \vdots & \vdots & \vdots & \vdots & \vdots & \vdots & \vdots & \vdots & \vdots \\ x_n & y_n & 1 & 0 & 0 & 0 & -x'_n x_n & -x'_n y_n & -x'_n \\ 0 & 0 & 0 & x_n & y_n & 1 & -y'_n x_n & -y'_n y_n & -y'_n \end{bmatrix}$$

Solution via SVD: Compute the SVD of \(\mathbf{A}\), \(\mathbf{A}=\mathbf{U}\mathbf{\Sigma}\mathbf{V}^\top\). The homography parameter vector \(\mathbf{h}\) is the right singular vector corresponding to the smallest singular value (last column of \(\mathbf{V}\)). Reshape \(\mathbf{h}\) into \(3\times 3\) to obtain \(\mathbf{H}\), then normalize so that \(H_{33}=1\).

Part A.3: Warp the Images

Inverse Warping Implementation

Nearest Neighbor Interpolation

Uses inverse warping to avoid holes in the output image. For each output pixel, we compute the corresponding source pixel location using the inverse homography, then round to the nearest integer coordinates to sample the source image.

Bilinear Interpolation

Similar to nearest neighbor but uses weighted averaging of the four neighboring pixels in the source image. The weights are computed based on the fractional part of the transformed coordinates, resulting in smoother output images.

Interpolation Comparison

Nearest Neighbor
  • Pros: Fast computation, preserves sharp edges
  • Cons: Blocky artifacts, aliasing
  • Best for: Binary images, when speed is critical
Bilinear
  • Pros: Smooth results, reduces aliasing
  • Cons: Slightly slower, can blur sharp edges
  • Best for: Natural images, when quality is important

Part A.4: Blend Images into a Mosaic

Blending Algorithm

Alpha Mask Generation

Alpha masks are generated using distance transforms to create smooth falloff from the center of each image to its edges. This ensures that pixels near the center of each image have higher weights, while edge pixels have lower weights.

Weighted Blending

The final mosaic is created using normalized weighted averaging. Each pixel in the output is computed as the weighted sum of all contributing input pixels, divided by the sum of weights. This creates smooth transitions in overlap regions.

Complete Procedure Explanation

The mosaic creation process follows these systematic steps:

  1. Image Capture: Take overlapping photographs from a fixed center of projection, rotating the camera between shots to ensure projective transformations.
  2. Point Selection: Manually select 4 corresponding feature points in both images using distinctive landmarks like corners, intersections, and high-contrast edges.
  3. Homography Computation: Set up an overdetermined linear system from the point correspondences and solve using SVD to find the 3Ã—3 homography matrix.
  4. Canvas Size Calculation: Determine the output mosaic dimensions by transforming the corners of both images and computing the union of their bounding boxes.
  5. Image Warping: Apply inverse warping to transform the second image into the coordinate system of the first image using either nearest neighbor or bilinear interpolation.
  6. Alpha Mask Generation: Create distance-based alpha masks for each image to ensure smooth blending, with higher weights at image centers and gradual falloff toward edges.
  7. Weighted Blending: Combine the warped images using normalized weighted averaging.

Part B: Feature Matching for Autostitching

In Part A, we manually selected corresponding points between images to compute homographies. Part B automates this process by detecting, describing, and matching features automatically using the Harris corner detector, feature descriptors, and robust matching techniques.

Part B.1: Harris Corner Detection

Harris Corner Detector

The Harris corner detector identifies interest points by analyzing the local image structure.

Harris Corners Detection Result

Harris corners overlaid on the bouldering scene. Red dots indicate detected corner points.

Adaptive Non-Maximal Suppression (ANMS)

ANMS reduces the number of detected corners by selecting a spatially distributed subset. For each corner, it computes the minimum suppression radius - the distance to the nearest stronger corner. Only corners with large suppression radii are retained, ensuring good spatial distribution across the image.

ANMS Result

Selected corners after ANMS. Green dots show the spatially distributed subset of corners.

Part B.2: Feature Descriptor Extraction

8x8 Feature Descriptors

For each detected corner, we extract an 8Ã—8 feature descriptor by sampling from a larger 40Ã—40 window around the corner. Each descriptor is bias/gain normalized to have zero mean and unit variance for robust matching.

Example Feature Descriptors

Sample 8Ã—8 normalized feature descriptors extracted from detected corners.

Part B.3: Feature Matching

Lowe's Ratio Test

Feature matching uses Lowe's ratio test to identify reliable correspondences. For each feature in the first image, we find its two nearest neighbors in the second image based on descriptor distance (SSD). A match is accepted only if the ratio of distances to the first and second nearest neighbors is below a threshold (typically 0.4-0.6), indicating a distinctive match.

Part B.4: RANSAC for Robust Homography

4-Point RANSAC Implementation

RANSAC (Random Sample Consensus) robustly estimates homographies in the presence of outliers. The algorithm repeatedly samples 4 random feature correspondences, computes a homography, and counts inliers (correspondences that agree with this homography within a threshold). The homography with the most inliers is selected as the final estimate.
